# Contributing to Chronicle Stack

Chronicle Stackへの貢献を検討していただき、ありがとうございます。

この文書は、貢献時の基本方針、ライセンス、商用ライセンスとの関係を説明します。

## 基本方針

Chronicle Stack v0.2.0 以降は AGPL-3.0-or-later で提供されます。

プロジェクトに貢献する場合、あなたの貢献物は、Chronicle Stack本体と同じくAGPL-3.0-or-laterで提供されることに同意したものとして扱います。

## 商用ライセンスとの関係

Chronicle Stackは、AGPL版に加えて、ZYX Corp株式会社による商用ライセンスを別途提供する場合があります。

そのため、商用ライセンス版にも含める必要がある大きな外部貢献については、マージ前に追加の合意またはContributor License Agreement（CLA）が必要になる場合があります。

正式なCLAまたはDCO方針は、[Contributor License Policy](docs/contributor-license-policy.md) に従います。

## 暫定CLA方針

正式なCLAテンプレートが整備されるまで、次の方針で運用します。

- 小さな修正、ドキュメント修正、誤字修正、テスト補足はAGPL-3.0-or-laterへの貢献として受け付けます。
- 新機能、大きなリファクタリング、データモデル変更、永続化形式変更などの大きな貢献は、Issueで事前相談してください。
- 商用ライセンス版にも含める必要がある貢献については、マージ前にZYX Corp株式会社との個別合意またはCLAが必要になる場合があります。
- CLAが必要と判断される貢献は、CLA整備前にはマージを保留する場合があります。

## 貢献時に確認すること

貢献者は、以下を確認してください。

- 自分がその貢献物を提供する権利を持っていること
- 第三者の秘密情報や非公開コードを含めていないこと
- 依存ライブラリや外部コードのライセンスに問題がないこと
- 生成AIを使った場合、その出力を自分の責任で確認していること
- Chronicle Stackのライセンス方針と矛盾しないこと

## Pull Request方針

Pull Requestでは、次を明記してください。

- 変更内容
- 目的
- テスト結果
- 既存仕様への影響
- RDE観点で保存されたもの、変換されたもの、未解決事項、逸脱リスク

## 大きな変更

次の変更は、事前にIssueで相談してください。

- データモデル変更
- JSONL形式変更
- ライセンスや権利関係に関わる変更
- 外部サービス連携
- 新しい永続化方式
- GraphRAGやDashboardなど大きな機能追加

## 法的注意

この文書は貢献方針の説明であり、法的助言ではありません。正式な契約条件が必要な場合は、個別に確認してください。
