# Chronicle Stack AGPL遵守ガイド

この文書は、Chronicle Stack を AGPL-3.0-or-later のもとで利用する場合の考え方を説明します。

この文書は法的助言ではありません。正確な条件は `LICENSE`、AGPL本文、個別契約を確認してください。

## 基本方針

Chronicle Stack v0.2.0 以降は AGPL-3.0-or-later で提供されます。

AGPL版を利用する場合、利用、改変、配布、ネットワークサービスとしての提供は、AGPLの条件に従う必要があります。

## そのまま利用する場合

Chronicle Stackをそのまま取得し、ローカルで利用する場合、通常はAGPLの条件に従って利用できます。

配布する場合は、ライセンス表示、著作権表示、対応するソースコードへのアクセスを維持してください。

## 改変して利用する場合

Chronicle Stackを改変する場合、改変内容を含む対応ソースコードをAGPL条件に従って提供できるようにしてください。

改変版を配布する場合だけでなく、ネットワークサービスとして利用者に提供する場合にも注意が必要です。

## SaaSまたはネットワークサービスとして提供する場合

Chronicle Stackを改変してネットワークサービスとして提供する場合、利用者が対応するソースコードへアクセスできるようにする必要があります。

この点が、AGPLを採用する主な理由です。Chronicle Stackは、文脈、判断、出所を扱う基盤であり、改変された文脈基盤が閉じたサービスとして囲い込まれないことを重視します。

## Docker imageやバイナリ配布の場合

Docker image、バイナリ、パッケージとして配布する場合、対応するソースコードへのアクセス方法を明示してください。

配布物にライセンス表示と著作権表示を含めることも必要です。

## 商用利用について

AGPLは商用利用そのものを禁止するものではありません。

ただし、閉源組み込み、独自SaaS提供、AGPLとは異なる条件での利用を希望する場合は、ZYX Corp株式会社との個別契約により商用ライセンスを提供する場合があります。

## 商用ライセンスが必要になりうる例

- Chronicle Stackを改変し、対応ソースコードを公開せずにSaaSとして提供したい場合
- 閉源製品にChronicle Stackを組み込みたい場合
- 顧客向け製品として再配布したい場合
- AGPLとは異なる保証、補償、サポート条件が必要な場合
- OEMまたはEnterprise契約が必要な場合

## 公式版との混同を避ける

forkや改変版を公開する場合は、公式版と混同されないようにしてください。

名称利用については [商標・名称利用ポリシー](trademark-policy.md) を参照してください。
