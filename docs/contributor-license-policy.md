# Chronicle Stack Contributor License Policy

この文書は、Chronicle Stack への外部貢献とライセンス処理の暫定方針を説明します。

この文書は法的助言ではありません。正式なContributor License Agreement（CLA）や契約条件は、必要に応じて専門家の確認を受けます。

## 基本方針

Chronicle Stack v0.2.0 以降は AGPL-3.0-or-later で提供されます。

通常の貢献は、Chronicle Stack本体と同じく AGPL-3.0-or-later で提供されることを前提に受け付けます。

ただし、Chronicle Stack は商用ライセンスを別途提供する場合があります。そのため、商用ライセンス版にも含める可能性がある大きな外部貢献については、事前に追加の合意が必要です。

## 暫定方針

正式なCLAテンプレートが整備されるまで、次の方針で運用します。

1. 小さな修正、ドキュメント修正、誤字修正、テスト補足はAGPL-3.0-or-laterへの貢献として受け付けます。
2. 新機能、大きなリファクタリング、データモデル変更、永続化形式変更などの大きな貢献は、Issueで事前相談してください。
3. 商用ライセンス版にも含める必要がある貢献については、マージ前にZYX Corp株式会社との個別合意またはCLAが必要になる場合があります。
4. CLAが必要と判断される貢献は、CLA整備前にはマージを保留する場合があります。

## DCOについて

Developer Certificate of Origin（DCO）は軽量な貢献確認手段です。

ただし、Chronicle Stackでは商用ライセンスを別途提供する可能性があるため、DCOのみで十分かどうかは未決定です。

今後、DCOを採用するか、個人CLA・法人CLAを採用するかを別途決定します。

## CLAが必要になりうる例

- 新しい主要機能
- データモデルの変更
- JSONL形式やIndex形式の変更
- 外部サービス連携
- GraphRAG、Dashboard、サーバー機能などの大きな追加
- 商用ライセンス版にも含める必要があるコード

## 貢献者の確認事項

貢献者は、少なくとも以下を確認してください。

- 自分がその貢献物を提供する権利を持っていること
- 第三者の秘密情報や非公開コードを含めていないこと
- 依存ライブラリや外部コードのライセンスに問題がないこと
- 生成AIを使った場合、その出力を自分の責任で確認していること
- Chronicle Stackのライセンス方針と矛盾しないこと

## 今後の方針
