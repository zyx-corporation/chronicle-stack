# Graph Export Inspection Commands

Chronicle Stack v0.4 adds read-only graph inspection commands for the derived `graph-json` structure.

These commands inspect the same deterministic graph-ready export generated by:

```bash
chronicle export --format graph-json
```

They do not implement GraphRAG, embeddings, vector search, graph database storage, or semantic retrieval.

## Commands

```bash
chronicle-graph summary
chronicle-graph summary --json
chronicle-graph nodes
chronicle-graph nodes --type context
chronicle-graph nodes --json
chronicle-graph edges
chronicle-graph edges --type chronicle_has_event
chronicle-graph edges --json
```

## Summary

`summary` prints total node and edge counts, plus counts by node type and edge type.

```bash
chronicle-graph summary
```

Machine-readable output:

```bash
chronicle-graph summary --json
```

## Nodes

```bash
chronicle-graph nodes
chronicle-graph nodes --type context
chronicle-graph nodes --json
```

`--type` filters by `node_type`.

## Edges

```bash
chronicle-graph edges
chronicle-graph edges --type chronicle_has_event
chronicle-graph edges --json
```

`--type` filters by `edge_type`.

## Contract

- Read-only.
- JSONL is not mutated.
- Indexes are not rebuilt.
- Output is derived from the current graph export.
- Human output is flexible.
- JSON output is intended for inspection and scripting.

## Non-goals

- No GraphRAG query engine.
- No embeddings.
- No vector database.
- No graph database.
- No LLM calls.
- No graph mutation API.
- No semantic retrieval.

## Note

The v0.4 implementation exposes these commands as `chronicle-graph` rather than nesting them under `chronicle graph`. This keeps the primary `chronicle` CLI stable while adding graph inspection as an auxiliary console command.
