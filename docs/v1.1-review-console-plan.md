## Purpose

Chronicle Stack v1.0.0 is a stable local-first CLI foundation. v1.1 begins the human-facing GUI / review console track without weakening the no-daemon, no-server, no-external-runtime boundary.

The first v1.1 implementation step is to extend the existing static HTML dashboard into a read-first review console.

This document records the static review-console starting point, not the final UI destination.
For the target-state final product UI, use the final UI design specifications as the current forward-looking reference.

## Product boundary

The v1.1 review console is:

- a single-file static HTML export
- generated explicitly by `chronicle export --format html`
- read-only
- local-first
- inspectable
- suitable for human review of Chronicle state

The v1.1 review console is not:

- a daemon
- a web server
- a hosted app
- a cloud sync surface
- a model API client
- a GraphRAG engine
- a vector DB or graph DB UI
- an enforcement mechanism
- a correctness proof system

## MVP implemented in this track

The initial v1.1 read-first console adds:

- Review Console boundary panel
- canonical local path panel for primary records, audit log, lifecycle log, and packages
- package review snapshot
- package review findings table
- audit events section
- lifecycle markers section
- review-oriented summary cards
- navigation anchors for review sections
- filtering across review findings, audit events, lifecycle markers, and existing record rows

## Why static first

A static HTML enhancement is the safest first GUI step because it preserves the v1.0 architecture:

- no background process
- no browser-exposed local server
- no persistent UI state
- no write operations
- no new runtime dependency
- no hidden import of Sayane or CSG-RAG runtime responsibilities

This is not the final GUI vision. It is the smallest safe step from CLI-first foundation toward human-facing review.

## Future candidate tracks

Future v1.1+ issues may consider:

- `chronicle ui` as an explicitly launched local web UI
- a Tauri companion app
- richer timeline grouping
- linked detail panes for context / artifact / decision / RDE records
- package review drill-downs
- side-by-side source provenance inspection
- explicit write-capable actions with preview and confirmation

Write-capable GUI actions should remain out of the first read-first console.

## Acceptance criteria

- Existing HTML export becomes a review console without requiring a server.
- Audit and lifecycle events are visible in the HTML export.
- A package review snapshot is visible in the HTML export.
- The console contains explicit no-daemon / no-server / no-model-runtime / no-GraphRAG-runtime boundary language.
- Tests cover the new review console sections.

## Warning classification

- Adoption warning: CLI-only v1.0.0 is technically releasable but insufficient for broad non-developer usability.
- Runtime warning: GUI convenience must not silently introduce daemon/server/cloud dependencies.
- Semantics warning: GUI review states must not be mistaken for enforcement or correctness proof.
- Architecture warning: review console must not collapse Chronicle Stack into Sayane, CSG-RAG, or a hosted app.

## RDE review

### Preserved

- v1.0.0 remains a valid stable local-first CLI foundation.
- Static export remains read-only and derived.
- CLI and local files remain canonical.
- Advisory/diagnostic semantics remain explicit.

### Transformed

- The existing dashboard becomes a review console oriented toward human inspection.

### Supplemented

- Review Console boundary panel.
- Package review snapshot.
- Audit/lifecycle review panels.
- GUI adoption framing.

### Unresolved

- Whether a future interactive GUI should live in Chronicle Stack core or a companion repository.
- Whether the first interactive UI should be Tauri or explicit local web UI.
- Which write-capable actions, if any, belong in a later GUI.

### Deviation risks

- Overcorrecting GUI weakness by adding hidden runtime services.
- Treating a read-only console as enforcement or proof.
- Narrowing Chronicle Stack into a single downstream application UX.
