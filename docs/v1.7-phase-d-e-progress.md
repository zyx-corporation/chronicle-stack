# v1.7 Phase D/E Progress

この文書は、local AI runtime vertical slice に向かう v1.7 のうち、Phase D/E 相当として repo 上で進んだ実装面をまとめます。

## Scope

- Phase D: vector / graph adapter contracts with local file-backed placeholder indexes
- Phase E 相当の一部: read-only UI visibility for the placeholder index surface

## Added surfaces

CLI:

- `chronicle ai-index status`
- `chronicle ai-index vector add`
- `chronicle ai-index vector search`
- `chronicle ai-index graph add-node`
- `chronicle ai-index graph add-edge`
- `chronicle ai-index graph neighbors`

Storage:

- `.chronicle/ai_indexes/vector_index.json`
- `.chronicle/ai_indexes/graph_index.json`

Read-only UI:

- `/api/ai-index-status`
- `/api/ai-index-vector`
- `/api/ai-index-graph-nodes`
- `/api/ai-index-graph-edges`

## Boundary

- No LLM invoked
- No embedding provider invoked
- No vector DB invoked
- No graph DB invoked
- No GraphRAG runtime invoked
- No external services
- Indexes are local file-backed placeholder derived surfaces
- Search result is assistive, not correctness proof
- Primary Chronicle records remain authoritative
- Read-only UI visibility does not add GUI mutation

## Notes

vector search は本格 embedding ではなく、token overlap / substring による placeholder scoring です。

graph surface は local adjacency です。query engine や graph database ではありません。

今回の UI 可視化は read-only visibility に留まり、approve / reject / request-changes のような GUI mutation は含みません。
