## Issue breakdown

### #TBD v1.7: Define local reviewer identity and session boundary for gated GUI mutation

Scope:

- define what reviewer identity fields are required when the local mutation gate is enabled
- define how session labels are represented and validated
- clarify what is descriptive metadata versus what is required for mutation acceptance

Out of scope:

- multi-user authentication
- hosted identity providers
- authorization enforcement beyond local single-operator assumptions

### #TBD v1.7: Refine mutation-readiness blockers and enablement criteria

Scope:

- make `mutation_readiness` blockers operationally precise
- document explicit prerequisites for local write enablement
- keep overview, list, and detail readiness wording aligned

Out of scope:

- enabling writes by default
- replacing CLI as the primary write path

### #TBD v1.7: Align blocked-route preview and CLI fallback contracts across all local UI surfaces

Scope:

- keep blocked preview, apply response, recovery command, and follow-up command contracts synchronized
- ensure review queue, summary jobs, and detail views share the same fail-closed vocabulary
- extend smoke coverage only where contract surfaces expand

Out of scope:

- real auth/authz enforcement
- new mutation capability beyond the existing explicit gate

### #TBD v1.7: ADR for browser-triggered review write semantics and audit insertion

Scope:

- define browser-triggered decision semantics before broader GUI mutation work
- define where audit insertion occurs relative to Chronicle append
- define fail-closed expectations for partial failures

Out of scope:

- implementation of all write routes
- multi-user workflow design

### #TBD v1.7: Design explicit GUI review write routes and failure contracts

Scope:

- define route shapes for approve / reject / request-changes
- define request/response contracts for successful and failed local writes
- define how explicit local enablement is checked server-side

Out of scope:

- default-on mutation
- unaudited write paths

### #TBD v1.7: Define browser-triggered rollback and error classification model

Scope:

- classify audit-write failure versus Chronicle append failure
- define rollback reporting rules for browser-triggered review actions
- align error classes with CLI/operator recovery guidance

Out of scope:

- proving distributed correctness
- cross-process transactional guarantees beyond the current local boundary

### #TBD v1.7: Add explicit manual provider invocation beyond invoke-plan dry-run

Scope:

- add explicit manual provider execution under current no-hidden-runtime rules
- preserve explicit provider kind / model / invocation mode / external-call flags
- keep execution opt-in and operator-visible

Out of scope:

- hidden background execution
- automatic invocation on record creation

### #TBD v1.7: Persist provider-produced draft or pending-review outputs with provenance

Scope:

- define storage shape for provider-produced draft outputs
- keep generated output derived, reviewable, and non-authoritative until accepted
- preserve provenance and runtime metadata in persisted records

Out of scope:

- automatic approval
- generated output as primary record

### #TBD v1.7: Expand retrieval-plan composition without GraphRAG runtime claims

Scope:

- deepen retrieval-plan composition beyond the current placeholder skeleton
- preserve explicit dry-run semantics
- keep language and docs clear that this is not yet a GraphRAG engine

Out of scope:

- vector DB runtime
- graph DB runtime
- ranking quality claims

### #TBD v1.7: Refine downstream handoff and package handoff review contracts

Scope:

- improve downstream handoff visibility where it helps operator review
- refine package handoff preview and package review snapshot contracts
- keep handoff outputs clearly read-only or derived until explicitly persisted

Out of scope:

- hidden export automation
- package mutation side effects without explicit operator action
