# Chronicle Stack v0.8 Package Review Workflow

Status: active technical workflow  
Theme: Verified Package / Export Review Workflow

## Purpose

v0.8 adds a local review step before package persistence or handoff.

The review step inspects a controlled integration package and returns:

- `pass`
- `warning`
- `blocked`

This is a diagnostic workflow. It does not call external runtimes and does not certify semantic correctness.

## Primary command

```bash
chronicle package review --purpose "review" --target local
```

To review selected Context records:

```bash
chronicle package review \
  --purpose "review" \
  --target local \
  --context ctx_example
```

To review a persisted package:

```bash
chronicle package review --package pkg_example
```

JSON output:

```bash
chronicle package review --purpose "review" --target local --json
```

## Status meaning

`pass` means no package-review findings were produced.

`warning` means the package has advisory findings. Example: an unclassified Context.

`blocked` means the package has a finding that should stop handoff until reviewed. Example: sensitive Context content targeted at an external environment without explicit policy permission.

## Review inputs

The review workflow uses the existing controlled package builder. It observes:

- Context classification metadata
- package warnings
- record-level warnings
- prompt-like markers detected by the package builder
- lifecycle exclusions already reflected in package warnings
- target environment

## Boundary

Package review is local and diagnostic.

It does not:

- call models
- call GraphRAG engines
- use vector databases
- use graph databases
- run a server
- run a daemon
- certify correctness

## Relationship to v0.7

v0.7 made classification, audit, lifecycle, and doctor remediation operational.

v0.8 uses those surfaces to add a review checkpoint before package persistence or handoff.

## RDE review
